╔══════════════════════════════════════════════════════════════════════════════╗
║                         SPOT INSTANCE BEHAVIOR                               ║
╚══════════════════════════════════════════════════════════════════════════════╝

  Normal operation:
  ┌─────────────────────────────────────────────────────┐
  │ Your instance is running (2–3 sec response time)   │
  └─────────────────────────────────────────────────────┘

  Spot interruption (rare, 1–2x/month):
  ┌──────────────────────────────────────────────────────┐
  │ 14:23:10  AWS: "We need this capacity back"         │
  │           Instance metadata endpoint returns timeout │
  │ 14:23:15  CloudWatch detects unhealthy instance     │
  │ 14:23:16  ASG launches replacement (from template)  │
  │ 14:25:00  New instance boots, user_data.sh runs     │
  │ 14:26:00  Ollama models cached (fast), app starts   │
  │ 14:26:30  ✓ Back online                             │
  │                                                      │
  │ User experience: 3 min of 404 errors (retries), then works
  │ Acceptable for MVP; unacceptable for production     │
  └──────────────────────────────────────────────────────┘

  Resolution: ASG capacity_rebalance = true → auto-recovery

║                             IMPLEMENTATION NOTES                             ║
╚══════════════════════════════════════════════════════════════════════════════╝

  • Ollama model (llama3.1:8b-instruct-q4_0) is ~15 GB; cached in EC2 ephemeral
  • First startup takes ~10 min (model download); subsequent restarts ~1 min
  • Medicine data (654 remedies, 222k rubric entries) pre-loaded into RDS + vectors
  • JWT tokens expire in 24 hours (configurable in application.yml)
  • CloudWatch logs retained for 7 days (configurable)
  • No WAF, no multi-AZ, no cross-region (MVP optimizations for cost)
